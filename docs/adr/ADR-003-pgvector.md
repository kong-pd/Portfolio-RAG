# ADR-003 · 向量存储：pgvector 同库，不引入独立向量库

**状态**：Accepted（2026-07-10）

**背景**：需要向量检索；同时有大量关系数据（用户、文档、会话）。

**选项**：Qdrant / Milvus / Weaviate 专用向量库；Elasticsearch；Postgres + pgvector。

**决定**：pgvector + HNSW 索引。关系数据与向量同一事务域——删除文档级联清理向量（FR-D5）天然一致，无跨库对账；运维面减半；本项目规模（远低于百万向量）下检索性能足够。

**后果**：HNSW 索引维度上限约 2000，这直接约束了 ADR-008 的 embedding 维度选择；ANN 极限性能不及专用库，属已知且可接受。
**重审触发**：向量规模达数百万级，或需要专用库级别的复杂过滤性能。
