## 3. 追溯与几点说明

**FR 覆盖**：FR-01 → US-01/02；FR-02 → US-03；FR-04 → US-04/05；FR-05 → US-07；FR-06 → US-08/09；FR-07 → US-10。

**FR-03（向量化存储）有意不立 story**：它是纯基础设施能力，对用户不可见、无独立用户价值，其验收完全由 US-03 / US-04 的 AC（AC-03.x）携带。Story 描述价值，FR 描述能力，二者不必一一对应——这是刻意的建模选择，不是遗漏。

**US-06 单独立项**：数据隔离本是横切约束，散在各 story 里容易被「顺手通过」。立为独立红线 story，意味着 D4 要做一次专项对抗验证（构造 A / B 用户重叠语料），它也是本项目相对 ChatPDF 类产品的核心卖点，值得一张独立卡片。

**看板建议**：四列 `Backlog → Doing → Review → Done`。Review 列的动作就是对照《02》逐条勾 AC；Doing 列同时不超过 1 张卡（单人项目，并行即自欺）。

**与熔断规则的联动**（《01》§3）：触发熔断时 US-07 ~ US-10 整列回 Backlog，US-04 的流式表现降级为一次性返回，其余 P0 故事不动。
